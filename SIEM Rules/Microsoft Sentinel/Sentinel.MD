# EntramoxReconciler – Microsoft Sentinel Detection Guide (KQL)

## Overview

This guide provides **Microsoft Sentinel detections (KQL)** to monitor identity lifecycle and privilege events generated by **entramoxreconciler** (Entra ID → Linux/Proxmox reconciliation).

**V2.5 additions:** Tiered accounts, Proxmox Backup Server (PBS), Proxmox Datacenter Manager (PDM), structured JSON audit log, remote API errors.

Use these detections to create **Scheduled Analytics Rules** for:

* Local user creation / deletion
* Account lock / unlock
* Sudo granted / revoked (group + sudoers file)
* Tiered (privileged) account provisioning and removal
* Users entering deletion grace period
* Proxmox VE / PBS / PDM user enable / disable + metadata updates
* Remote API authentication failures

---

## Data Sources

Your Linux host forwards logs to Sentinel via one of:

| Option | Table | Message field | Host field |
|--------|-------|--------------|------------|
| Syslog (AMA) | `Syslog` | `SyslogMessage` | `Computer` |
| CEF | `CommonSecurityLog` | `Message` | `DeviceName` |
| Custom | `Entramox_CL` | `RawData` | varies |

Replace `TABLE_HERE` and `MESSAGE_FIELD` throughout with your chosen values.

### Two log files to forward

| File | Format | Content |
|------|--------|---------|
| `/var/log/entramoxreconciler/thelog.log` | Text | All operational events |
| `/var/log/entramoxreconciler/audit.log` | JSON-lines | Security audit events (v2.5+) |

For the JSON audit log, ingest into a custom table (e.g. `EntramoxAudit_CL`) using the AMA Custom JSON connector, or use the Log Analytics DCR pipeline.

```kql
| where MESSAGE_FIELD has "entramoxreconciler"
```

## 1) Local User Created

```kql
TABLE_HERE
| where MESSAGE_FIELD has "Created local user:"
| extend User = extract(@"Created local user:\s+(\S+)", 1, MESSAGE_FIELD)
| extend Host = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, User, MESSAGE_FIELD
| summarize Count=count(), CreatedAt=max(TimeGenerated) by Host, User
| extend AccountCustomEntity=User, HostCustomEntity=Host
```

**Severity:** Medium | **Tactic:** Persistence

## 6) Tiered Account Created (v2.5)

Detect creation of a privileged tiered account (e.g. `a-john`) for a superadmin user.

```kql
TABLE_HERE
| where MESSAGE_FIELD has "Created tiered account"
| extend TieredUser = extract(@"Created tiered account '([^']+)'", 1, MESSAGE_FIELD)
| extend BaseUser   = extract(@"for base user '([^']+)'", 1, MESSAGE_FIELD)
| extend Host = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, TieredUser, BaseUser, MESSAGE_FIELD
| summarize Count=count(), CreatedAt=max(TimeGenerated) by Host, TieredUser, BaseUser
| extend AccountCustomEntity=TieredUser, HostCustomEntity=Host
```

**Severity:** High | **Tactic:** PrivilegeEscalation (T1548), Persistence (T1136.001)

> A new tiered account means a new privileged login has been provisioned. Always verify this aligns with the Entra superadmin group membership and expected change control.

## 8) PVE User Enabled / Disabled

**Severity:** Enable = High, Disable = Medium

## 9) PBS User Provisioned / State Changed (v2.5)

```kql
TABLE_HERE

    and (MESSAGE_FIELD has "created:" or MESSAGE_FIELD has "updated:" or MESSAGE_FIELD has "set enable=")

    MESSAGE_FIELD has "created:", "provisioned",

    Enabled == "0", "disabled",

| project TimeGenerated, Host, UserId, Action, MESSAGE_FIELD
| summarize Count=count(), LastSeen=max(TimeGenerated) by Host, UserId, Action
| extend HostCustomEntity=Host
```

**Severity:** Provisioned/Enabled = Medium, Disabled = Low

## 11) Remote API Auth Failure — PBS / PDM (v2.5)

Detects 401/403 responses from PBS or PDM, indicating credential or permission problems.

```kql
TABLE_HERE
| where MESSAGE_FIELD has "Remote API HTTP"
    and (MESSAGE_FIELD has " 401:" or MESSAGE_FIELD has " 403:")
| extend Method = extract(@"Remote API HTTP\s+(\S+)", 1, MESSAGE_FIELD)
| extend Url    = extract(@"Remote API HTTP\s+\S+\s+(\S+)\s+\d+", 1, MESSAGE_FIELD)
| extend Status = extract(@"(\d{3}):", 1, MESSAGE_FIELD)
| extend Host   = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, Method, Url, Status, MESSAGE_FIELD
| summarize Count=count(), LastSeen=max(TimeGenerated) by Host, Url, Status
| extend HostCustomEntity=Host
```

**Severity:** High | **Tactic:** CredentialAccess (T1552)

> Repeated 401s suggest the PBS/PDM API token has expired, been revoked, or is misconfigured.

## 15) Integrity Checker — Script Checksum Mismatch (v2.5)

Fires when `entramox_check.sh` detects the reconciler binary hash has changed.

```kql
TABLE_HERE
| where MESSAGE_FIELD has "[entramox_check]"
    and MESSAGE_FIELD has "script checksum mismatch"
| extend Path       = extract(@"file=(\S+)", 1, MESSAGE_FIELD)
| extend HaveHash   = extract(@"have=([0-9a-f]+)", 1, MESSAGE_FIELD)
| extend ExpectHash = extract(@"expect=([0-9a-f]+)", 1, MESSAGE_FIELD)
| extend Host = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, Path, HaveHash, ExpectHash, MESSAGE_FIELD
| summarize Count=count(), FirstSeen=min(TimeGenerated), LastSeen=max(TimeGenerated) by Host, Path
| extend HostCustomEntity=Host
```

**Severity:** Critical | **Tactic:** DefenseEvasion (T1195 — supply chain / binary swap)

> A mismatch means the script on disk no longer matches the hash stored at install time. Treat as a critical incident until ruled out.

## 16) Integrity Checker — File Tampering (Symlink / Owner / Permissions) (v2.5)

```kql
TABLE_HERE
| where MESSAGE_FIELD has "[entramox_check]"
    and (
        MESSAGE_FIELD has "symlink refused"
        or MESSAGE_FIELD has "file not owned by root"
        or MESSAGE_FIELD has "permissions too broad"
    )
| extend CheckType = case(
    MESSAGE_FIELD has "symlink refused",        "symlink_refused",
    MESSAGE_FIELD has "file not owned by root", "wrong_owner",
    MESSAGE_FIELD has "permissions too broad",  "perms_too_broad",
    "unknown"
)
| extend Path = extract(@"path=(\S+)", 1, MESSAGE_FIELD)
| extend Detail = case(
    MESSAGE_FIELD has "uid=",  strcat("uid=", extract(@"uid=(\S+)", 1, MESSAGE_FIELD)),
    MESSAGE_FIELD has "mode=", strcat("mode=", extract(@"mode=(\S+)", 1, MESSAGE_FIELD)),
    ""
)
| extend Host = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, CheckType, Path, Detail, MESSAGE_FIELD
| summarize Count=count(), LastSeen=max(TimeGenerated) by Host, CheckType, Path
| extend HostCustomEntity=Host
```

**Severity:** Critical | **Tactic:** DefenseEvasion / Persistence (T1574.010, T1222)

## 17) Integrity Checker — Env / Key File Hash Changed (v2.5)

```kql
TABLE_HERE
| where MESSAGE_FIELD has "[entramox_check]"
    and (MESSAGE_FIELD has "env checksum changed" or MESSAGE_FIELD has "key checksum changed")
| extend FileType   = case(
    MESSAGE_FIELD has "env checksum", "env_file",
    MESSAGE_FIELD has "key checksum", "key_file",
    "unknown"
)
| extend Path       = extract(@"path=(\S+)", 1, MESSAGE_FIELD)
| extend HaveHash   = extract(@"have=([0-9a-f]+)", 1, MESSAGE_FIELD)
| extend ExpectHash = extract(@"expect=([0-9a-f]+)", 1, MESSAGE_FIELD)
| extend Host = coalesce(tostring(Computer), tostring(HostName), tostring(DeviceName))
| project TimeGenerated, Host, FileType, Path, HaveHash, ExpectHash, MESSAGE_FIELD
| summarize Count=count(), LastSeen=max(TimeGenerated) by Host, FileType, Path
| extend HostCustomEntity=Host
```

**Severity:** High | **Tactic:** CredentialAccess (T1552.001)

> This fires as a WARNING in the log — could be a legitimate reconfiguration. Correlate with change-control records.

## Recommended Rule Configuration

| Rule | Frequency | Lookback | Severity |
|------|-----------|----------|----------|
| User created | 5 min | 10 min | Medium |
| User deleted | 5 min | 10 min | High |
| Sudo granted | 5 min | 10 min | High |
| Tiered account created | 5 min | 10 min | High |
| Tiered account removed | 5 min | 10 min | Medium |
| Remote API 401/403 | 5 min | 10 min | High |
| Auth credentials missing | 5 min | 10 min | Critical |
| PBS/PDM user provisioned | 15 min | 30 min | Medium |
| **Script checksum mismatch** | **5 min** | **10 min** | **Critical** |
| **File tampering (symlink/owner/perms)** | **5 min** | **10 min** | **Critical** |
| **Env/key hash changed** | **5 min** | **10 min** | **High** |
| **Missing monitored file** | **15 min** | **30 min** | **High** |

**Entity mapping:** Account → `AccountCustomEntity`, Host → `HostCustomEntity`

## Quick Start Checklist

1. Confirm log table + field names in your environment.
2. Replace `TABLE_HERE` and `MESSAGE_FIELD` in every query.
3. If using the JSON audit log, configure a DCR to ingest `audit.log` into `EntramoxAudit_CL`.
4. Create Scheduled Analytics Rules for detections 1–12 from the main log.
5. Add detections 13–14 if the JSON audit log is ingested.
6. Add detections 15–18 for integrity checker coverage (Critical/High severity).
7. Split sudo_granted (rule 5) into its own High-severity rule.
8. Set up entity grouping by `Host + User` for incident correlation.
